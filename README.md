В общем, как это работает:

Вначале прогоняем **сырые данные** через `dqc_analys`, в нем можем чуть поднастроить по случаю пару переменных.

Дальше получаем `issues_summary.json`, в котором в поле `suggested_action`, где хотим сделать какое-то действие, должны вручную вставить вариант действия, проанализировав данные, которые там предоставлены (варианты лежат в `action_notes.txt`).

И в конце, слой ETL по JSON-файлу: делают очистку и сохраняют всё в `clean_data`.
